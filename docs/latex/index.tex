\href{github.com/aviii06/vivid}{\texttt{ Vivid}} is an Open\+GL-\/based rendering framework providing an intuitive 2D and 3D rendering API.\hypertarget{index_autotoc_md1}{}\doxysection{Features}\label{index_autotoc_md1}
{\bfseries{Shader-\/\+Based Rendering}}\+: The renderer utilizes the power of shaders to achieve real-\/time rendering with support for vertex, fragment, and geometry shaders. This allows for flexible and efficient customization of the rendering pipeline.

{\bfseries{Basic Shapes}}\+: Easily render basic geometric shapes like cubes, spheres, cylinders, and planes. These primitives can be manipulated and combined to create complex scenes.

{\bfseries{Texture Mapping}}\+: Apply textures to objects for more realistic and visually appealing results. Experiment with different materials and lighting setups.

{\bfseries{\mbox{\hyperlink{class_camera}{Camera}} Controls}}\+: The renderer includes a camera system that allows you to navigate and explore the 3D environment using mouse and keyboard controls.

{\bfseries{Lighting}}\+: Implement point-\/light lighting models.

{\bfseries{Model Loading}}\+: Import 3D obj models from external files to populate your scenes with intricate designs.

{\bfseries{User Interface}}\+: The renderer comes with a simple user interface that provides controls for adjusting rendering settings, camera parameters, and more.\hypertarget{index_autotoc_md2}{}\doxysection{Integration}\label{index_autotoc_md2}

Once the renderer is running, feel free to explore the various features it offers. Modify shaders, adjust camera settings, load different models, and experiment with lighting configurations.\hypertarget{index_autotoc_md20}{}\doxysection{API}\label{index_autotoc_md20}
\hypertarget{index_autotoc_md21}{}\doxysubsection{Philosophy}\label{index_autotoc_md21}
The API is inspired primarily by \href{https://github.com/processing}{\texttt{ processing foundation}}.

You have a Rendering Interface which comes with 3 main functions\+: {\ttfamily Setup()}, {\ttfamily Draw()}, and {\ttfamily Im\+Gui\+Render()}. You can create a new class which extends to \mbox{\hyperlink{class_rendering_interface}{Rendering\+Interface}} and override these functions. This provides a multitude of different rendering APIs all with different levels of control to ease.\hypertarget{index_autotoc_md22}{}\doxysubsection{Types of API}\label{index_autotoc_md22}
\hypertarget{index_autotoc_md23}{}\doxysubsubsection{ECS API}\label{index_autotoc_md23}
This API provides a complete entity-\/component-\/based architecture. You can initialise an entity and give it many components and they all get rendered automatically. \hypertarget{index_autotoc_md24}{}\doxysubsubsection{Rendering 2\+D/3\+D APIs}\label{index_autotoc_md24}
These APIs create a more processing-\/like feel to rendering. You call a {\ttfamily \mbox{\hyperlink{class_vivid_1_1_renderer2_d_a5fdeecbc6c3d8b341c24f3096e308a97}{Vivid\+::\+Renderer2\+D\+::\+Init()}};} or {\ttfamily Vivid\+::\+Renderer3\+D\+::\+Init();} and can call many functions like {\ttfamily Vivid\+::\+Renderer2\+D\+::\+Draw\+Line(point1, point2, thickness, color);} and more. This makes it easy to create a small simulation which is not overly complicated.

\hypertarget{index_autotoc_md25}{}\doxysubsubsection{Mesh API}\label{index_autotoc_md25}
This is useful for small renders. You can simply create a Mesh object from an Obj file and then render it out by calling the {\ttfamily Draw(camera)} method. ~\newline

\hypertarget{index_autotoc_md27}{}\doxysubsubsection{Setup()}\label{index_autotoc_md27}
The {\ttfamily Setup()} function runs once and is useful for initializing your variables. The first line of the setup is reserved for your open\+GL configs. You can write your own configs or use one of the premade configs. For 3D it is recommended to use\+: {\ttfamily OPENGL\+\_\+\+CONFS} And for 2D\+: {\ttfamily OPENGL\+\_\+2\+D\+\_\+\+CONFS}\hypertarget{index_autotoc_md28}{}\doxysubsubsection{Draw()}\label{index_autotoc_md28}
This function is called every frame. And is not inside the Im\+GUI context.\hypertarget{index_autotoc_md29}{}\doxysubsubsection{Im\+Gui\+Render()}\label{index_autotoc_md29}
This function is similar to {\ttfamily Draw()} but is inside Im\+GUI context so this could be used to create custom UI elements.

To see examples using these APIs you can look in the examples folder in the root directory of the repo.\hypertarget{index_autotoc_md30}{}\doxysubsection{2D}\label{index_autotoc_md30}
First in your {\ttfamily Setup()} function use the macro {\ttfamily OPENGL\+\_\+2\+D\+\_\+\+CONFS}. To get started with 2D API you need to call {\ttfamily \mbox{\hyperlink{class_vivid_1_1_renderer2_d_a5fdeecbc6c3d8b341c24f3096e308a97}{Vivid\+::\+Renderer2\+D\+::\+Init()}};} in your {\ttfamily Setup()} function. This initializes the 2D renderer.

In your {\ttfamily Draw()} loop call {\ttfamily \mbox{\hyperlink{class_vivid_1_1_renderer2_d_a1198ec2ca56dabd29346ddd58cb7f0cd}{Vivid\+::\+Renderer2\+D\+::\+Begin\+Scene()}};} and once all the Draw\+Calls have been made call {\ttfamily \mbox{\hyperlink{class_vivid_1_1_renderer2_d_a653686fc23a3aa2f6e0a7ccad1ea6bb2}{Vivid\+::\+Renderer2\+D\+::\+End\+Scene()}};} In between these two functions, you can use the Renderer2D API to render anything. This all gets Batch\+Rendered for optimal performance.

Examples can be found in {\ttfamily examples/examples2D}.
